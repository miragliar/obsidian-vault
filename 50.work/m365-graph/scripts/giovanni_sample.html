Power BI Desktop ↔ Claude Desktop verbinden (MCP)

Erprobter Weg: Standalone-.exe aus der VS-Code-Extension – kein Node.js nötig.

Voraussetzungen: Windows · Power BI Desktop · Claude Desktop · Visual Studio Code (nur als Lieferant der Server-.exe).

Schritt 1 – VS Code: Extension installieren

  1. VS Code öffnen, beim Begrüßungsscreen „Continue without Signing In" (GitHub-Login nur für Copilot – nicht nötig).
  2. Links Extensions (Strg+Shift+X)
  3. Suchen: Power BI Modeling MCP (Publisher: analysis-services / Microsoft) → Install

Schritt 2 – Pfad zur Server-.exe finden

PowerShell öffnen und ausführen:

Get-ChildItem "$env:USERPROFILE\.vscode\extensions" -Recurse -Filter "powerbi-modeling-mcp.exe" | Select-Object -ExpandProperty FullName

Bei mehreren Versionen die höchste nehmen, z. B.:
C:\Users\<user>\.vscode\extensions\analysis-services.powerbi-modeling-mcp-0.5.8-win32-x64\server\powerbi-modeling-mcp.exe

Schritt 3 – Claude Desktop konfigurieren

Claude Desktop → Settings → Developer → Edit Config (Datei: %APPDATA%\Claude\claude_desktop_config.json). Eintrag einfügen – Pfad mit doppelten Backslashes \\!

Fall A – Datei leer / nur {}:

{
  "mcpServers": {
    "powerbi-modeling-mcp": {
      "type": "stdio",
      "command": "C:\\Users\\<user>\\.vscode\\extensions\\analysis-services.powerbi-modeling-mcp-0.5.8-win32-x64\\server\\powerbi-modeling-mcp.exe",
      "args": ["--start"]
    }
  }
}

Fall B – Datei hat schon Inhalt (z. B. preferences): mcpServers als eigenen Top-Level-Eintrag ergänzen und mit Komma abtrennen.

Häufigster Fehler: nur EINE öffnende { ganz oben; nach dem mcpServers-Block ein Komma, wenn danach noch etwas kommt; alle Backslashes doppelt (\\). Dann mit Strg+S speichern.

Schritt 4 – Claude neu starten

Claude über das Tray-Icon komplett beenden (nicht nur Fenster schließen) und neu öffnen. → Im Chat erscheint das Tool-/🔌-Symbol mit powerbi-modeling-mcp.

Schritt 5 – Verbinden & testen

  1. Power BI Desktop mit der .pbix öffnen und offen lassen.
  2. In Claude: „Connect to '<Dateiname>' in Power BI Desktop" (erkennt die lokale Instanz automatisch).
  3. Test: „Liste alle Tabellen und Beziehungen meines Modells auf."
